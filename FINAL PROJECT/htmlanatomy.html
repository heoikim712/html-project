Anatomy of an HTML

In this page, I'll be talking about how an HTML5 file is structured and explain it's anatomy.

Anatomy

Mainly, there are 2 main parts in an HTML5. There is the <head> section and a <body> section. The biggest differences of
these two parts is that the <head> section isn't visible at all (the only exception is the icon) and on the contrary, the <body>
section is the visible part of the web. In the <head> section you can see stuff like metadata, title, and other information that
you won't really see, but just because you can't see the contents in the <head> section, it doesn't mean that its irrelevant or
unimportant. In the <body> section, users will be able to actually physically see the content that is enbeded inside that section.
Contents include things like, text, digits, tables, lists, and other multimedia (images, audio, video, etc).

Elements in HTML5

Elements in HTML are kind of like sentences on an essay, essay being the HTML5. Elements are mainly separated into three
parts, the opening tag, the closing tag and the content that the developer decides to add to the page. Let me explain you the
three main parts of an HTML5 Element.

To start off, let's talk about Opening Tags and Closing Tags. Opening Tags will define the type of meaning the content inside
will have. Also all tags will be wraped with closing bracets (< and >). Opening tags can also contain Attributes, and attributes
are kinda like complementary information that will specify more specific information inside the Opening Tag. Values can also
appear after html attributes. Attribute values are the values assigned to attributes within HTML elements. These values provide
additional information about an element or specify how an element should behave. The type and purpose of the attribute
determine what kind of value it can accept.

Some examples of attributes can be like, <a href = "https://youtube.complementary"> Click to go to Youtube </a>.
In this example you can see that the underlined part of the code is the HTML Attribute and the highlighted section is the Value
of the attribute.

HTML Tags

Tags are also a huge part of an HTML, there are a wide range of tags that have different meanings depending on their names.
Some examples of HTML tags can be:

  1. <nav>= Navigation Tag
  2. <hr>= Horizontal Rule Tag
  3. <figure>= Figure Tag
  4. <mark>= Highlighting Tag
  5. <form>= Form Tag
  6. <li>= List Tag
  7. <table>= Table Tag

Inline and Block Level Elements

There are generally two types of Elements in an HTML archive. Inline Elements and Block Elements. This could be a bit confusing
so hang on tight.

1.) Inline Elements

Inline elements behave differently in terms of display. They do not introduce line breaks or start on a new line; instead, they flow
within the content horizontally, taking up only as much width as required for their content. The primary purpose of inline elements
is to style and format specific portions of text within block-level elements. Examples of inline elements include anchor links (<a>),
text emphasis ( <em>), strong emphasis (<em>), inline images (<img>), and span elements (<span>). Unlike block-level elements,
inline elements cannot contain other block-level elements but can nest other inline elements within them.

1.) Block Elements

Unlike Inline Elements, Block-level elements in HTML create a distinct "block" or rectangular box within the layout of a web page.
They typically initiate a new line in the content and expand to occupy the full width available, effectively pushing any adjacent
content below them. These elements are commonly used to establish the fundamental structure of a web page. Examples of block-
level elements include headings (e.g., <h1>, <h2>), paragraphs (<p>), lists (<ul>, <ol>), tables (<table>), and divisions (<div>).
Block-level elements offer versatility in organizing content because they can nest both block-level and inline elements within them.
This characteristic makes them a foundational component in creating structured and organized web documents.
