Forms and Inputs=

Forms and Inputs in an HTML file is a very different topic compared with the previous things we've talked about. All the previous topics were about the devs sending content or information to the users of the website. But in today's lesson, in Forms and Inputs, the user is the one that sends information to the dev or the creator of the website. In the following paragraphs I will expand more upon this topic. Enjoy today's lesson!

Elements we'll use=

Here's a list of elements you'll need to create a Form and an Input.

<form>
This tag is used for creating a form element.
<label> Tag=
This tag show the user that the input is about. So if you are waiting for the user to type in, lets say, their first name, then you'll type firstname (an example) in the <label> tag.
<input>
In this tag, you'll have to type in the type of input you want the audience to type in. There are lost of types of input types, one of the basic ones is the Text Type. The text input is used so that the user will type in a text input for the author of the website.

Attributes we'll use=

We also need to know some Attributes. These are=

1.) Attributes inside <form> tags

First, I'll explain you the Attributes for the <form> tag. There are 2 tags I'll talk about, the "action" and "method" tags

The "action" attribute is used in an HTML form element to specify the URL (Uniform Resource Locator) to which the data submitted through the form should be sent for processing when the user submits the form.

The other attribute is called "Method". This attribute specifies the HTTP request method that should be used when submitting the form data to the server. The two most commonly used HTTP methods with the "method" attribute are "GET" and "POST," but other methods like "PUT" and "DELETE" can also be used.

2.) Attribute inside <label> tag

Next, I'll explain you the Attributes for the <label> tag. The first one is the attribute "for". This attribute is used to indicate that the label stands for a specific input. 3.) Attributes inside <input>

type
This attribute is used to specify the type of input the user is going to type down. Some examples for the possible values for this attribute are, text, reset, submit, checkbox, and date.
id
As I mentioned in the "attributes inside the label tag", this attribute is used to tell the <label> tag to stand for the "id" of the next <input>
name
This attribute is simply used to name the input that is entered by the client.
placeholder
This attribute is used to show the audience a bit of context and give the client a bit of an idea of what is expected to be typed in.

One thing that I want to say is that there are more attributes I can talk about, but I haven't told you guys because those attributes are for other types of <input> tags. One example can be "min" and "max".
